# Academic Figure Prompt Generator

为科研论文和学术汇报生成高质量、**完整自包含**的插图绘制Prompt。

## 核心原则

1. **完整自包含**：生成的Prompt必须让生图AI无需任何额外信息即可正确生成
2. **技术准确**：公式、参数、文字必须正确表达学术/技术含义
3. **矢量图风格**：扁平化、简洁、专业，避免写实/炫酷风格
4. **无元信息污染**：坐标、区域编号、尺寸标注等辅助信息不出现在图中

## 关键规范

### 避免元信息污染
生成Prompt时，以下内容仅作为描述指引，**不应出现在最终图像中**：
- 坐标位置（如"位置(3cm, 2cm)"）
- 区域编号（如"Zone A"、"区域1"）
- 尺寸标注（如"宽2cm×高3cm"）
- 内部注释（如"此处放置..."）

正确做法：用自然语言描述相对位置关系（如"左上角"、"中心偏右"、"下方"）

### 神经网络详细绘制
当图中涉及神经网络、深度学习模块时，**必须详细描述网络结构**，而非简单图标：
- 详见 `references/neural-network-drawing.md`

## Phase 1: 交互式信息收集

根据用户已提供的信息智能跳过已知项，用简洁的方式询问关键信息。

### 必问问题

**Q1: 图片类型与布局风格**
> 请选择最接近您需求的类型：
> 1. **框架流程图** - 模块化、左右/上下流水线布局
> 2. **系统场景图** - 展示网络拓扑、设备部署、空间关系
> 3. **算法架构图** - 神经网络、强化学习、数据流
> 4. **时序/阶段图** - 时间轴、多阶段流程
> 5. **2.5D/等轴测图** - 立体城市、网络场景
> 6. **环绕/辐射图** - 中心场景+周围模块
> 7. **对比/并列图** - 多方案对比
> 8. **其他** - 请描述或提供参考图

**Q2: 核心内容**
> 请简述图片需要表达的内容，或上传相关文档/参考图。

### 按需询问

**Q3: 画布比例**（若未指定）
> 比例偏好：16:9（宽屏）/ 4:3（传统）/ 1:1（方形）/ 其他

**Q4: 视觉风格**（若未指定）
> - 配色：学术蓝灰 / 多彩清新 / 柔和渐变 / 高对比黑白 / 自定义
> - 维度：纯2D扁平 / 2.5D等轴测 / 轻微立体感
> - 图标风格：线性图标 / 填充图标 / 简化剪影

**Q5: 语言**
> 图内文字：中文 / 英文 / 双语

**Q6: 特殊需求**
> - 是否需要图例(Legend)
> - 是否需要图注(Caption)
> - 子图编号格式
> - 其他要求

## Phase 2: Prompt生成

### 输出要求：完整自包含

生成的Prompt必须满足：
1. **自包含**：生图AI无需查看原文档或额外解释
2. **技术准确**：所有公式、变量、参数名称正确无误
3. **视觉完整**：每个需要显示的元素都有明确描述
4. **无歧义**：使用具体描述而非模糊指代

### 文字/公式/参数处理规范

**必须逐一检查并明确列出**：
- 图中需要显示的所有文字标签
- 数学公式（写出完整LaTeX或Unicode形式）
- 变量名和参数符号
- 模块名称和说明文字

**格式要求**：
```
图中文字清单：
- 模块标题：[列出所有]
- 数学符号：$\alpha_t$, $\mathbf{h}_t^L$, $\pi_\theta$ 等
- 公式：$\mathcal{F}(a) = ...$（如需显示完整公式）
- 连接标注：[列出箭头上的标签]
- 图例文字：[如有]
```

### Prompt输出结构

```markdown
# [图片标题]

## 1. 整体设计
- **图片类型**：[具体类型]
- **视觉风格**：矢量扁平/2.5D等轴测/...，[配色风格]
- **画布比例**：[如16:9]
- **整体布局**：[自然语言描述布局结构]

## 2. 详细场景描述
[用完整的自然语言段落描述整个图片，包括：]
- 各区域/模块的位置关系（用相对位置词：左上、中央、右下等）
- 每个元素的外观（形状、颜色、大小比例）
- 元素之间的连接关系
- 特殊效果（阴影、渐变等）

[如涉及神经网络，必须详细描述网络结构，见neural-network-drawing.md]

## 3. 连接与数据流
[描述所有箭头、连线：]
- 箭头类型（实线/虚线/粗细）
- 箭头颜色
- 箭头上的标注文字

## 4. 图中文字清单
[逐一列出所有需要在图中显示的文字]
- 标题/标签：...
- 数学符号：...
- 公式（如有）：...
- 图例：...

## 5. 配色方案
[列出主要颜色及其用途]
- 模块A类型：[颜色名] [色号]
- 模块B类型：...
- 箭头/连线：...
- 背景：...

## 6. 重要约束
- 不要在图中显示：坐标数值、区域编号、尺寸标注
- 确保所有文字清晰可读
- [其他特定约束]

## 7. 图注（如需）
Figure X. [完整图注文字]
```

### 质量检查清单

生成Prompt后自检：
- [ ] 所有技术术语/公式是否准确？
- [ ] 神经网络结构是否详细描述（非简单图标）？
- [ ] 文字清单是否完整？
- [ ] 是否避免了元信息（坐标/编号）出现在图中？
- [ ] 描述是否足够具体，无模糊表达？

## Phase 3: 迭代优化

生成初版后询问：
- 布局是否符合预期
- 是否需要调整元素或添加细节
- 配色风格是否满意

### 推荐配色示例：马卡龙柔和系

适用于模块化框图、系统流程图、多区域架构图。特点：柔和低饱和度，区分度高，视觉舒适。

| 色号 | HEX | 用途建议 |
|------|-----|----------|
| Color 01 | #D2E0F7 | 浅蓝，主模块背景/数据处理区 |
| Color 02 | #E3ECD7 | 浅绿，输入/环境相关模块 |
| Color 03 | #FFE6D7 | 浅桃，中间层/数据流模块 |
| Color 04 | #E6B6AC | 粉珊瑚，性能指标/参数面板 |
| Color 05 | #F2E0EC | 浅粉紫，约束条件/辅助模块 |
| Color 06 | #FFF2CD | 浅黄，优化/决策相关区域 |
| Color 07 | #BC7F00 | 深金色，标题文字/强调边框 |

**使用建议**：
- 用不同浅色区分不同功能模块（如：预测模块用浅蓝#D2E0F7，优化模块用浅黄#FFF2CD）
- 虚线边框配合浅色填充，增强层次感
- 标题文字使用深金色#BC7F00，在浅色背景上清晰可读
- 各模块间连线使用灰色或对应模块的深色版本
