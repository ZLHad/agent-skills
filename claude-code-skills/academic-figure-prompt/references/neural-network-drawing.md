# Neural Network Drawing Guide 神经网络详细绘制指南

当图中涉及神经网络、深度学习、强化学习架构时，需要详细绘制网络结构，而非简单图标。

---

## 一、何时需要详细绘制

| 场景 | 处理方式 |
|------|----------|
| 网络是图的核心主题 | 必须详细绘制结构 |
| 网络是处理流程的一个环节 | 中等详细程度，展示关键层 |
| 网络仅作为示意（如"AI模型"） | 可用简化图标+标签 |

---

## 二、全连接网络 (Fully Connected / MLP)

### 详细绘制方式
```
描述模板：
一个[N]层全连接神经网络，采用2.5D或正面视角绘制：
- 输入层：[M]个圆形节点，垂直排列，颜色[X]
- 隐藏层1：[K]个圆形节点，与输入层全连接，连接线为浅灰色
- 隐藏层2：[J]个圆形节点...
- 输出层：[P]个圆形节点，颜色[Y]
- 层间标注：层名称或神经元数量
- 激活函数标注（可选）：在层旁标注"ReLU"、"Softmax"等
```

### 示例描述
> 一个4层MLP网络结构图。从左到右依次为：输入层（5个蓝色节点，标注"Input"）、隐藏层1（8个绿色节点，标注"Hidden 128"）、隐藏层2（8个绿色节点，标注"Hidden 64"）、输出层（3个橙色节点，标注"Output"）。相邻层之间用浅灰色细线全连接。整体呈现左窄右窄中间宽的结构。

## 三、卷积神经网络 (CNN)

### 详细绘制方式
```
描述模板：
CNN结构采用2.5D层叠视角绘制，展示特征图的空间维度变化：
- 输入：[H×W×C]的立方体/矩形块，标注尺寸
- Conv层：绘制为比输入略小的立方体，用不同颜色区分，标注"Conv 3×3"
- Pooling层：尺寸进一步缩小，可用不同颜色
- 多个卷积块之间用箭头连接
- 最后展平(Flatten)后接全连接层
- 在每个块下方或旁边标注：层类型、输出尺寸、通道数
```

### 示例描述
> 一个图像分类CNN结构，2.5D等轴测视角。从左到右：
> - 输入图像：蓝色立方体，标注"224×224×3"
> - Conv Block 1：两个堆叠的绿色立方体（略小），标注"Conv 3×3, 64"，后接浅蓝色薄片表示"MaxPool 2×2"
> - Conv Block 2：两个堆叠的黄色立方体（更小），标注"Conv 3×3, 128"，后接MaxPool
> - Conv Block 3：橙色立方体，标注"Conv 3×3, 256"
> - Flatten：用箭头指向一个垂直的长条表示展平
> - FC层：传统全连接网络表示，输出"1000 classes"
> 整体有轻微的3D透视感，块之间用箭头连接表示数据流向。

## 四、Transformer / Attention

### 详细绘制方式
```
描述模板：
Transformer Block采用模块化绘制：
- 整体框：圆角矩形包围整个Block
- Multi-Head Attention子模块：
  - 三个输入箭头标注Q、K、V
  - 内部显示多个attention head（可用多个平行小矩形表示）
  - 输出箭头
- Add & Norm：横条形状，标注"Add & Norm"或"LayerNorm"
- Feed Forward：矩形块，可展开显示两层Dense
- 残差连接：用弧线绕过模块连接到Add节点
- 多个Block堆叠时，用"×N"标注重复次数
```

### 示例描述
> Transformer Encoder结构图。绘制一个完整的Encoder Block：
> - 最外层：浅蓝色圆角矩形边框
> - 顶部"Multi-Head Attention"模块：内部显示4个平行的小矩形代表4个attention head，左侧三个输入箭头分别标注Q、K、V，右侧一个输出
> - 第一个"Add & Norm"：灰色横条，有一条弧线从Attention输入绕到这里表示残差连接
> - "Feed Forward Network"模块：标注"FFN"，可选显示内部Dense→ReLU→Dense
> - 第二个"Add & Norm"：同上，有残差连接
> - Block右上角标注"×6"表示堆叠6层

## 五、强化学习架构 (RL)

### Actor-Critic详细绘制
```
描述模板：
Actor-Critic架构，清晰展示：
- Environment模块：绿色边框，输出State/Observation、Reward
- Actor网络：橙色模块，详细绘制网络层结构（不只是方框）
  - 输入：状态s
  - 若干隐藏层
  - 输出：动作分布π(a|s)或确定性动作
- Critic网络：蓝色模块，详细绘制网络层结构
  - 输入：状态s（或状态-动作对）
  - 若干隐藏层
  - 输出：价值V(s)或Q(s,a)
- 数据流箭头：
  - 实线：前向传播/数据流
  - 虚线：梯度反传/参数更新
- 关键标注：loss函数、TD error、advantage等
```

### 示例描述
> PPO Actor-Critic架构图。
> - 左侧"Environment"：绿色虚线框，内含一个简化的环境示意（如网格世界），输出两个箭头分别标注"$s_t$"（状态）和"$r_t$"（奖励）
> - 中上方"Actor Network $\pi_\theta$"：橙色实线框，内部详细绘制3层全连接网络（输入层→128节点隐藏层→64节点隐藏层→输出层），输入接收状态$s_t$，输出动作概率分布，旁边标注"$\pi_\theta(a|s)$"
> - 中下方"Critic Network $V_\phi$"：蓝色实线框，内部详细绘制3层全连接网络结构，输入状态$s_t$，输出标量价值$V_\phi(s)$
> - 右侧数据流：
>   - Actor输出动作$a_t$回到Environment（实线箭头）
>   - Critic输出用于计算TD error的虚线流
> - 底部：显示损失函数"$L^{CLIP}$"和"$L^{VF}$"，用虚线箭头指向对应网络表示参数更新

## 六、2.5D绘制技巧

### 何时使用2.5D
- CNN特征图层叠
- 网络结构需要展示"深度"感
- 多个并行处理路径
- 复杂的数据流需要空间分离

### 2.5D描述要点
```
关键词：
- "等轴测视角"或"isometric view"
- "层叠立方体"表示CNN特征图
- "前后遮挡关系"明确层次
- "统一光源方向"保持一致性（通常左上方）
- "轻微阴影"增加立体感但不过度
```

## 七、网络结构+流程图结合

当神经网络是更大系统的一部分时：

### 描述策略
1. 先描述整体流程布局
2. 在涉及网络的位置，展开详细描述网络结构
3. 明确网络的输入来源和输出去向
4. 保持网络详图与整体流程的视觉一致性

### 示例
> 整体是一个从左到右的处理流程图。
> - 左侧"输入模块"：滑动窗口示意，内含5个时序数据块
> - 中间"特征提取"区域，包含两个并行分支：
>   - 上分支"长期通道"（蓝色背景区域）：
>     - 内部绘制一个Transformer Encoder结构（详细展示Multi-Head Attention和FFN层，非简单方框）
>     - 后接一个对比学习模块，显示两个增强视图和相似度计算
>   - 下分支"短期通道"（绿色背景区域）：
>     - 内部绘制一个1D-CNN结构（3个层叠的特征图块，尺寸递减）
> - 右侧"融合与决策"：
>   - 门控融合模块（显示加权求和符号）
>   - 策略网络（详细绘制一个小型MLP：3→64→32→1）

## 八、避免的问题

### ❌ 错误示例
- "神经网络图标"—— 过于简化，无法传达结构信息
- "一个方框写着NN"—— 缺乏细节
- "网络结构（略）"—— 未实际描述

### ✅ 正确做法
- 描述层数、每层节点数/通道数
- 描述层类型（Conv、FC、Attention等）
- 描述激活函数（如果关键）
- 描述数据维度变化
- 用颜色区分不同功能的层
