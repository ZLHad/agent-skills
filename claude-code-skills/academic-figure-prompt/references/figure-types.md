# Figure Types Guide 图片类型专项指南

本文件针对不同类型的学术图片提供设计指南和Prompt生成策略。

---

## 类型1: 系统场景图 (System Scenario)

### 特征
- 展示网络拓扑、设备部署、空间层次关系
- 包含多种实体：卫星、无人机、基站、用户、车辆、建筑等
- 强调覆盖区域、通信链路、数据流向

### 布局策略
- **垂直分层**：天空层（卫星/无人机）→ 空中层（UAV）→ 地面层（基站/用户）
- **水平分区**：不同地理区域（城市/乡村/海洋）
- **覆盖区域**：用虚线椭圆或渐变圆表示服务范围

### Prompt要点
```
描述时注意：
1. 明确层次结构和空间关系
2. 区分不同类型的通信链路（星间/星地/地面）
3. 使用图例说明箭头/线条含义
4. 标注关键节点和区域名称
```

### 示例描述片段
> 场景为空天地一体化网络。天空层有3颗LEO卫星呈三角分布，卫星间用绿色虚线连接表示星间链路。中间层有2架无人机悬停于城市上空，用黄色闪电符号表示与地面的无线连接。地面层分为左右两个区域：左侧城市区域包含基站塔、多栋建筑、移动车辆；右侧郊区包含农田和分散用户。整体用虚线大框包围表示网络覆盖范围。

## 类型2: 2.5D等轴测场景图 (Isometric Scene)

### 特征
- 伪3D视角，呈现空间立体感
- 常用于展示智慧城市、网络部署、系统架构
- 建筑、街道、设备采用等轴测绘制

### 布局策略
- **视角**：俯视45°等轴测视角
- **层次**：底层地面/建筑 → 中层设备/车辆 → 上层无人机/信号
- **深度**：通过遮挡关系和大小渐变表现

### Prompt要点
```
描述时注意：
1. 明确说明"2.5D等轴测视角"或"isometric style"
2. 描述建筑、道路的立体表现方式
3. 注意前后遮挡关系
4. 使用一致的光影方向（通常左上光源）
```

### 示例描述片段
> 采用2.5D等轴测视角的智慧城市场景。画面中心是一组城市建筑群，包含高层办公楼、住宅楼、学校等，建筑采用简化几何风格，有轻微立体感和统一的左上方阴影。道路呈网格分布，有小汽车和行人。城市上空悬停3架无人机，用虚线圆表示其覆盖范围。画面右上角有一颗卫星与无人机通信。左侧有一个放大的细节框展示边缘服务器部署。

## 类型3: 算法架构图 (Algorithm Architecture)

### 特征
- 展示神经网络结构、强化学习框架、数据处理流水线
- 包含Actor-Critic、Encoder-Decoder、Attention等模块
- 强调数据流向和模块间交互

### 布局策略
- **流水线型**：左→右或上→下的数据处理流程
- **双塔型**：Actor和Critic并列
- **编解码型**：对称的Encoder-Decoder结构
- **多智能体**：多个并列的Agent模块

### Prompt要点
```
描述时注意：
1. 明确模块的输入输出关系
2. 区分不同类型的模块（用颜色/形状）
3. 标注关键变量和公式
4. 用箭头明确数据流向
5. 损失函数和更新路径用虚线或不同颜色
```

### 示例描述片段
> Actor-Critic强化学习架构图。左侧是Environment模块（绿色边框），输出观测$o_t$和奖励$r_t$。中间上方是Actor网络（橙色模块），接收观测输出动作$a_t$；下方是Critic网络（蓝色模块），接收观测输出价值$V(s_t)$。右侧是Experience Replay缓冲区（灰色圆柱）。用实线箭头表示前向传播，虚线箭头表示反向传播/梯度更新。底部标注损失函数。

## 类型4: 时序/阶段流程图 (Timeline/Phase Diagram)

### 特征
- 展示时间维度上的多阶段处理流程
- 通常有明确的时间轴或阶段划分
- 不同时间尺度的操作可能交替进行

### 布局策略
- **水平时间轴**：顶部或中部放置时间轴
- **垂直下探**：时间轴下方展开各阶段详情
- **分区背景**：不同阶段用不同背景色区分

### Prompt要点
```
描述时注意：
1. 明确时间轴的起止和刻度
2. 标注关键时间点或事件
3. 用颜色区分大时间尺度和小时间尺度
4. 阶段内的详情模块与时间轴对齐
```

### 示例描述片段
> 双时间尺度优化流程图。顶部是水平时间轴，从左到右标注"Beginning"到"Time Slot"。时间轴上有大刻度（Communication Round）和小刻度（Time Slot）。大刻度对应浅绿色背景的"大尺度缓存优化"阶段，小刻度对应浅橙色背景的"小尺度任务卸载"阶段。每个阶段下方展开详细的处理流程模块。

## 类型5: 环绕/辐射布局图 (Radial Layout)

### 特征
- 中心是核心场景或系统
- 周围环绕多个功能模块或应用场景
- 强调中心与周围的关联关系

### 布局策略
- **中心焦点**：核心场景/系统占据画面中心
- **辐射模块**：4-8个模块环绕中心
- **连接线**：从中心向外辐射或用弧形连接

### Prompt要点
```
描述时注意：
1. 中心内容要足够吸引视觉焦点
2. 周围模块大小和样式保持一致
3. 连接线清晰但不杂乱
4. 每个周围模块包含图标+文字说明
```

### 示例描述片段
> 中心是一个椭圆形的空天地一体化网络场景（包含卫星、无人机、基站、城市建筑的综合示意）。周围环绕6个粉色/黄色圆角矩形模块，分别标注："1.资源分配"、"2.信道建模"、"3.图像处理"、"4.安全增强"、"5.语义通信"、"6.智能部署"。每个模块内有简短描述和小图标。用橙色弧形箭头从中心指向各模块。顶部有大标题"Generative AI-enabled SAGIN"。

## 类型6: 多层架构图 (Multi-layer Architecture)

### 特征
- 展示Cloud-Edge-User或类似的多层系统架构
- 每层包含不同类型的节点和服务
- 强调层间通信和服务部署

### 布局策略
- **水平分层**：顶层Cloud → 中层Edge → 底层User
- **虚线框分组**：每层用虚线大框包围
- **连接线**：层间用虚线/实线连接，层内用实线

### Prompt要点
```
描述时注意：
1. 每层用不同背景色或边框样式区分
2. 层标签放在左侧或框内
3. 节点图标在层内分布合理
4. 层间连接避免交叉混乱
```

### 示例描述片段
> 三层边缘计算架构图。顶层"Cloud"：深蓝色虚线框，包含云数据中心图标和多个彩色方块表示的服务模型。中层"Edge"：黄色虚线框，包含3组基站+服务器组合，每组旁边有对应的服务方块。底层"User"：绿色虚线框，每个基站下方有椭圆形用户群。Cloud到Edge用虚线连接，Edge到User用闪电符号表示无线连接。

## 类型7: 对比/并列布局图 (Comparison Layout)

### 特征
- 展示多个方案、方法或场景的对比
- 强调差异和相似性
- 常用于Before/After、Baseline/Proposed等对比

### 布局策略
- **左右并列**：两个方案左右对比
- **上下对比**：优化前后上下对比
- **网格布局**：多个方案矩阵排列

### Prompt要点
```
描述时注意：
1. 保持对比双方的结构一致性
2. 用颜色或标注突出差异点
3. 必要时添加对比箭头或VS符号
4. 统一的图例说明
```

---

## 类型8: 混合型复杂图 (Hybrid Complex)

### 特征
- 结合多种类型元素
- 通常包含：场景示意 + 算法流程 + 数据流
- 需要合理的空间划分

### 布局策略
- **主次分明**：确定视觉焦点区域
- **模块化分区**：不同类型内容用边框分隔
- **统一风格**：虽然内容复杂，但保持视觉一致性

### Prompt要点
```
描述时注意：
1. 先描述整体布局划分
2. 再逐区域详细描述
3. 明确区域间的连接关系
4. 使用一致的配色和图标风格
```

## 通用设计原则

### 视觉风格
- **矢量扁平**：简洁几何形状，避免复杂纹理
- **适度立体**：可用轻微阴影增加层次，但不要过度3D
- **一致性**：同类元素使用统一的图标风格

### 配色原则
- **主色不超过3种**：1主色 + 1-2辅助色
- **功能色分明**：不同类型模块用不同颜色
- **对比适度**：重要元素突出，次要元素弱化

### 标注原则
- **关键变量必标**：输入输出、核心参数
- **避免堆砌**：不是所有元素都需要标注
- **字体统一**：正文/标题/变量用统一字体族
