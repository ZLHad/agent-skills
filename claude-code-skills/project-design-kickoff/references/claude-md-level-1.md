# CLAUDE.md — Level 1 轻量模板

适合个人小项目、POC、单脚本探索。强调"讨论过程沉淀"，不涉及复杂文档体系。

---

## 模板内容

```markdown
# {项目名称} — 项目规范

## 项目概述

{用户提供的一句话描述}

当前阶段：**{阶段}**。

## 协作模式

本项目采用**对话式迭代设计**——通过多轮讨论逐步明确思路。每次会话中有价值的思考必须沉淀为文档。

### DesignNotes/ 是设计讨论的主阵地

1. 每次会话产出一份文档：`日期_session-NN_主题.md`
2. 讨论过程中有重要结论时就及时写入，不要等到最后
3. 新 session 可以"修正"旧 session 的结论，但**不改旧文**
4. 讨论中涌现的想法要即时捕获——用户说出一个有价值的观点时，立即记录到当前 session 文档

### 会话末尾主动写 session doc 的触发条件（满足任一即写）

- 本次对话产生了重大设计决策或认知修正
- 完成了重要功能、模块的修改或讨论
- 用户主动说"记录一下这次会话"

不要等用户催——这是 Claude 的主动责任。

### 每次会话开始时

1. 读取 `DesignNotes/` 下最近的文档，了解讨论进展
2. 读取项目 memory（如有），恢复上下文

### 跨 session 的认知升级

讨论产生对项目方向的重大决策或认知变化时，同步更新项目 memory。**DesignNotes 记录过程，memory 记录结论**。

## 目录结构

```
{项目名称}/
├── CLAUDE.md              ← 本文件，协作规范
├── DesignNotes/           ← 按 session 追加的讨论记录
└── (待建) {根据项目需要}
```

## 语言与风格

- {语言偏好}
- 设计文档要**有观点、有判断**，不要面面俱到地罗列
- 代码和技术方案用**段落式说明**，不堆 bullet point
```

## 使用说明

- 项目名称、描述、阶段、语言偏好从 Phase 1 收集
- 目录结构按项目实际情况微调
- **不要加**：doc/ 双轨规范、架构约束、参考管理——那是 Level 2/3 的事
