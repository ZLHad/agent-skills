# Knowledge Base Manager

## 核心功能

管理和整理本地知识库，将分散的单次对话记录转化为结构化、可检索、可追踪的个人知识体系。

## 工作流程

### 步骤1：参数配置与初始化

#### 1.1 路径配置
**首次使用**：
- 询问用户本地知识库路径
- 保存路径配置到 `/home/claude/.knowledge-base-config.json`
- 验证路径有效性

**后续使用**：
- 读取配置文件获取默认路径
- 显示当前路径，询问是否使用默认路径
- 如用户想修改，更新配置文件

**配置文件格式**：
```json
{
  "local_path": "/Users/xxx/我的知识库/",
  "last_organized_date": "2025-01-25T10:30:00Z",
  "last_organized_count": 12
}
```

#### 1.2 整理参数确认
询问用户（提供智能默认值）：

**整理模式**：
- 选项A：增量（只处理上次整理后的新文件）【默认】
- 选项B：全量（处理所有文件）
- 选项C：指定日期范围

**如果选择增量**：
- 显示上次整理时间（从配置文件读取）
- 确认："处理 [上次日期] 之后的新文件？"
- 可以让用户修改起始日期

**如果选择日期范围**：
- 询问起始日期（YYYY-MM-DD）
- 询问结束日期（YYYY-MM-DD，默认今天）

**类型筛选**（可选）：
- 全部类型【默认】
- 仅学术相关（学术写作 + 学术探讨）
- 仅学术写作
- 仅学术探讨
- 其他指定类型

**输出详细度**：
- 标准模式【默认】：包含摘要和详细报告
- 完整模式：额外生成所有专题整理（错误手册、模式库等）
- 快速模式：仅生成摘要

---

### 步骤2：文件扫描与解析

#### 2.1 扫描本地目录
```bash
# 使用bash_tool扫描目录
cd [用户配置的路径]
find . -name "knowledge-*.md" -type f
```

识别所有符合命名规范的知识库文件。

#### 2.2 读取并解析文件
对每个文件：
1. 使用 `view` 工具读取文件内容
2. 提取YAML元数据：
   - 提取日期 (extraction_date 或 date)
   - 对话类型 (conversation_type 或 type)
   - 主要话题 (main_topics 或 topics)
   - 重要程度 (importance_level)
   - 关键标签 (key_tags 或 tags)
3. 提取关键内容：
   - 执行摘要
   - 重点记忆清单（⭐⭐⭐、⚠️、💡 标记的内容）
   - 后续行动建议

#### 2.3 按条件筛选
根据用户选择的参数：
- 日期范围筛选
- 类型筛选
- 重要程度筛选（可选）

#### 2.4 分类统计
按以下维度分类：
- **按对话类型**：学术写作、学术探讨、编程开发、生活其他等
- **按主题领域**：论文写作、强化学习、深度学习等
- **按时间**：按周/月分组
- **按重要程度**：高/中/低

### 步骤3：智能分析与提取

#### 3.1 基础处理（所有类型）
对所有筛选出的文件：
- 统计文档数量、分布
- 提取高频主题和标签
- 生成时间序列视图
- 构建基础索引

#### 3.2 学术写作深度处理（重点）
如果包含学术写作类文档，深度处理：

**A. 错误模式分析**
- 提取所有 ⚠️ 标记的高频错误
- 按错误类型分类（语法/词汇/句式/逻辑/规范/风格）
- 统计每个错误的出现频次
- 追踪改正情况（早期vs近期对比）
- 识别已消失的错误 ✅
- 识别仍在出现的错误 ❌

**B. 写作模式提取**
- 提取所有可复用模板和成功模式
- 识别引言/方法/讨论/结论等不同章节的写作模式
- 合并相似模式
- 标注适用场景和成功案例

**C. 个人习惯档案**
- 汇总句式偏好
- 统计连接词使用模式
- 分析时态/语态把握情况
- 整理术语库

**D. 进步追踪**
- 对比早期（前1/3文档）vs 近期（后1/3文档）
- 计算错误率变化
- 识别技能提升
- 生成进步曲线

参考模板：`references/academic-writing-analysis.md`

#### 3.3 学术探讨深度处理（重点）
如果包含学术探讨类文档，深度处理：

**A. 概念网络构建**
- 提取所有讨论的核心概念
- 识别概念间的关系（依赖、对比、应用等）
- 按理解深度分层（基础/进阶/高级）
- 标注仍存在的疑问 ❓

**B. 思维框架提取**
- 识别可复用的分析框架
- 提取问题解决模板
- 归纳有效的思维方法
- 记录成功的类比和例子

**C. 认知演进追踪**
- 追踪对同一概念理解的深化过程
- 识别重要的认知突破点 💡
- 记录思维方式的转变 🔄

参考模板：`references/academic-discussion-analysis.md`

#### 3.4 其他类型基础处理
对编程、生活等其他类型：
- 提取关键收获
- 整理可复用资源（代码片段、工具、方法）
- 构建简单索引

**文档5：索引文件（必须）**

## 智能特性

### 1. 进度反馈
在处理大量文件时，提供进度反馈：
```
正在扫描文件... ✓
已发现 18 个知识库文件
正在解析文件... (6/18)
正在分析学术写作内容... ✓
正在构建索引... ✓
正在生成报告... ✓
```

### 2. 异常处理
- 如果路径不存在 → 提示并要求重新输入
- 如果没有找到任何文件 → 提示检查路径和文件命名
- 如果YAML解析失败 → 跳过该文件，记录警告
- 如果筛选后文档为0 → 提示调整筛选条件

### 3. 增量优化
- 如果是增量模式且没有新文件 → 提示"知识库已是最新"
- 如果专题文档已存在 → 采用"更新"模式（合并新内容）
- 如果检测到文件被修改 → 提示是否重新处理

### 4. 智能建议
基于分析结果：
- 如果错误率仍高 → 建议重点复习错误手册
- 如果某主题文档很多 → 建议考虑细分主题
- 如果某类型文档过少 → 建议平衡学习内容
- 如果学习强度下降 → 鼓励保持学习节奏

## 注意事项

1. **文件安全**：只读取，不修改原始knowledge-*.md文件
2. **增量效率**：增量模式下只处理必要文件，提高速度
3. **灵活性**：支持用户自定义筛选条件
4. **可追溯**：保留历史整理报告，可对比不同时期
5. **容错性**：个别文件解析失败不影响整体流程
6. **用户友好**：提供清晰的进度反馈和结果摘要

## 学术写作专题整理工作流

针对学术写作类原始记录，执行以下5阶段工作流：

```
1. 状态分析 → 2. 专题整理 → 3. 精简压缩 → 4. 逐一验证 → 5. 结构优化
```

### 阶段1：状态分析

读取主题索引，检查：
- 原始文档数量（已处理/待处理）
- 专题文档状态（已填充/部分填充/待填充）
- 文件命名是否规范

### 阶段2：专题整理（核心）

**双维度分类体系**：

| 章节规范 | 维度规范 |
|----------|----------|
| 摘要写作规范 | 语法规范 |
| 引言写作规范 | 术语用词规范 |
| Related-Work写作规范 | LaTeX与公式规范 |
| 系统模型写作规范 | 算法伪代码规范 |
| 方法与算法写作规范 | 逻辑与结构规范 |
| 实验与结论写作规范 | 标点与格式规范 |

**提取原则**：
- 提取"错误→正确"案例对比
- 标记贡献来源（便于追溯）
- 添加检查清单

### 阶段3：精简压缩

**四删原则**：
1. 删空白"待补充"章节
2. 删重复内容
3. 删冗长描述（改表格）
4. 保留核心案例对比

**目标**：压缩40%-70%，提升信息密度

### 阶段4：逐一验证

**逐篇核对**：
1. 遍历每篇原始文档
2. 列出其关键知识点
3. 核对专题文档中是否覆盖
4. 记录覆盖情况，遗漏则补充

**标准**：每篇文档覆盖率100%

### 阶段5：结构优化

1. **修命名**：统一 `knowledge-YYYYMMDD-type-topic.md`
2. **清归档**：删除旧版/临时文件
3. **同步索引**：主题/标签/时间/文件清单全部更新

### 自检清单

```
□ 双维度分类完成
□ 每个规范有贡献来源
□ 精简后无空白章节
□ 逐一验证覆盖率100%
□ 文件命名统一
□ 4个索引全部同步
```
