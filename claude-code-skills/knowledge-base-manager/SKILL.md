# Knowledge Base Manager

## 核心功能

管理和整理本地知识库，将分散的单次对话记录转化为结构化、可检索、可追踪的个人知识体系。

## 工作流程

### 步骤1：参数配置与初始化

#### 1.1 路径配置
**首次使用**：
- 询问用户本地知识库路径
- 保存路径配置到 `/home/claude/.knowledge-base-config.json`
- 验证路径有效性

**后续使用**：
- 读取配置文件获取默认路径
- 显示当前路径，询问是否使用默认路径
- 如用户想修改，更新配置文件

**配置文件格式**：
```json
{
  "local_path": "/Users/xxx/我的知识库/",
  "last_organized_date": "2025-01-25T10:30:00Z",
  "last_organized_count": 12
}
```

#### 1.2 整理参数确认
询问用户（提供智能默认值）：

**整理模式**：
- 选项A：增量（只处理上次整理后的新文件）【默认】
- 选项B：全量（处理所有文件）
- 选项C：指定日期范围

**如果选择增量**：
- 显示上次整理时间（从配置文件读取）
- 确认："处理 [上次日期] 之后的新文件？"
- 可以让用户修改起始日期

**如果选择日期范围**：
- 询问起始日期（YYYY-MM-DD）
- 询问结束日期（YYYY-MM-DD，默认今天）

**类型筛选**（可选）：
- 全部类型【默认】
- 仅学术相关（学术写作 + 学术探讨）
- 仅学术写作
- 仅学术探讨
- 其他指定类型

**输出详细度**：
- 标准模式【默认】：包含摘要和详细报告
- 完整模式：额外生成所有专题整理（错误手册、模式库等）
- 快速模式：仅生成摘要

---

### 步骤2：文件扫描与解析

#### 2.1 扫描本地目录
```bash
# 使用bash_tool扫描目录
cd [用户配置的路径]
find . -name "knowledge-*.md" -type f
```

识别所有符合命名规范的知识库文件。

#### 2.2 读取并解析文件
对每个文件：
1. 使用 `view` 工具读取文件内容
2. 提取YAML元数据：
   - 提取日期 (extraction_date 或 date)
   - 对话类型 (conversation_type 或 type)
   - 主要话题 (main_topics 或 topics)
   - 重要程度 (importance_level)
   - 关键标签 (key_tags 或 tags)
3. 提取关键内容：
   - 执行摘要
   - 重点记忆清单（⭐⭐⭐、⚠️、💡 标记的内容）
   - 后续行动建议

#### 2.3 按条件筛选
根据用户选择的参数：
- 日期范围筛选
- 类型筛选
- 重要程度筛选（可选）

#### 2.4 分类统计
按以下维度分类：
- **按对话类型**：学术写作、学术探讨、编程开发、生活其他等
- **按主题领域**：论文写作、强化学习、深度学习等
- **按时间**：按周/月分组
- **按重要程度**：高/中/低

---

### 步骤3：智能分析与提取

#### 3.1 基础处理（所有类型）
对所有筛选出的文件：
- 统计文档数量、分布
- 提取高频主题和标签
- 生成时间序列视图
- 构建基础索引

#### 3.2 学术写作深度处理（重点）
如果包含学术写作类文档，深度处理：

**A. 错误模式分析**
- 提取所有 ⚠️ 标记的高频错误
- 按错误类型分类（语法/词汇/句式/逻辑/规范/风格）
- 统计每个错误的出现频次
- 追踪改正情况（早期vs近期对比）
- 识别已消失的错误 ✅ 和仍在出现的错误 ❌

**B. 写作模式提取**
- 提取所有可复用模板和成功模式
- 识别不同章节的写作模式，合并相似模式
- 标注适用场景和成功案例

**C. 个人习惯档案**
- 汇总句式偏好、连接词使用模式
- 分析时态/语态把握情况，整理术语库

**D. 进步追踪**
- 对比早期（前1/3文档）vs 近期（后1/3文档）
- 计算错误率变化，识别技能提升

参考模板：`references/academic-writing-analysis.md`

#### 3.3 学术探讨深度处理（重点）
如果包含学术探讨类文档，深度处理：

**A. 概念网络构建**：提取核心概念，识别概念间关系，按理解深度分层

**B. 思维框架提取**：识别可复用的分析框架和问题解决模板

**C. 认知演进追踪**：追踪对同一概念理解的深化过程，识别认知突破点 💡

参考模板：`references/academic-discussion-analysis.md`

#### 3.4 其他类型基础处理
对编程、生活等其他类型：
- 提取关键收获
- 整理可复用资源（代码片段、工具、方法）
- 构建简单索引

---

### 步骤4：生成结构化输出

#### 4.1 目录结构与文件命名
在本地知识库根目录创建/更新结构化目录（原始记录、整理报告、专题整理、索引）。

详细目录结构和命名规范见：`references/output-file-specs.md`

#### 4.2 生成核心文档

**文档1：整理报告（必须）**
文件名：`整理报告-YYYYMMDD.md`，包含摘要、详细统计、文档清单三层结构。
参考模板：`templates/organizing-report-template.md`

**文档2：学术写作错误手册（如有学术写作内容）**
文件名：`专题整理/学术写作错误手册.md`，包含高频错误Top10及详细记录。
参考模板：`templates/writing-errors-handbook-template.md`

**文档3：学术写作模式库（如有学术写作内容）**
文件名：`专题整理/学术写作模式库.md`，包含各章节写作模式及适用场景。
参考模板：`templates/specialized-docs-templates.md`

**文档4：思维框架库（如有学术探讨内容）**
文件名：`专题整理/思维框架库.md`，包含可复用分析框架和问题解决模板。
参考模板：`templates/specialized-docs-templates.md`

**文档5：索引文件（必须）**
主题索引、标签索引、时间索引。
参考模板：`templates/index-templates.md`

#### 4.3 更新配置
更新 `/home/claude/.knowledge-base-config.json`：
- 记录本次整理时间
- 记录本次处理文档数量

---

### 步骤5：输出与交付

#### 5.1 展示摘要
向用户展示整理摘要（终端输出）：
```
✅ 知识库整理完成！

📊 本次统计：
- 扫描文档：18篇
- 处理文档：6篇（新增）
- 学术写作：4篇
- 学术探讨：2篇

📝 生成文档：
- 整理报告-20250130.md
- 学术写作错误手册.md（更新）
- 学术写作模式库.md（更新）
- 思维框架库.md（新增）
- 索引文件（全部更新）

💡 关键发现：
- 主谓一致错误仍在出现（需重点关注）
- 引言写作质量显著提升
- 新掌握2个思维框架

🔗 查看详细报告：[提供链接]
```

#### 5.2 提供文件下载
使用 `present_files` 提供关键文档下载：整理报告、更新的专题文档、索引文件。

#### 5.3 给出建议
基于分析结果，给出下一步学习建议、需要重点关注的问题、知识库维护建议。

## 触发方式与参数

### 基础触发
```
"整理知识库"  "管理本地知识"  "更新知识索引"  "整理学习记录"
```

### 带参数触发
```
"整理最近一周的知识"
"整理学术写作相关的内容"
"整理1月15日到1月30日的记录"
"全量整理知识库"
"生成学术写作错误手册"
```

### 参数解析规则
- 如提到"最近X天/周/月" → 自动计算日期范围
- 如提到"学术写作/论文" → 筛选类型为学术写作
- 如提到"全量/所有" → 设置为全量模式
- 如提到具体日期 → 使用日期范围模式
- 如提到"错误手册/模式库/框架" → 重点生成对应专题文档
