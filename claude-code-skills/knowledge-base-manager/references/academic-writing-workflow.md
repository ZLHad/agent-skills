# 学术写作专题整理工作流

针对学术写作类原始记录，执行以下5阶段工作流：

```
1. 状态分析 → 2. 专题整理 → 3. 精简压缩 → 4. 逐一验证 → 5. 结构优化
```

## 阶段1：状态分析

读取主题索引，检查：
- 原始文档数量（已处理/待处理）
- 专题文档状态（已填充/部分填充/待填充）
- 文件命名是否规范

## 阶段2：专题整理（核心）

**双维度分类体系**：

| 章节规范 | 维度规范 |
|----------|----------|
| 摘要写作规范 | 语法规范 |
| 引言写作规范 | 术语用词规范 |
| Related-Work写作规范 | LaTeX与公式规范 |
| 系统模型写作规范 | 算法伪代码规范 |
| 方法与算法写作规范 | 逻辑与结构规范 |
| 实验与结论写作规范 | 标点与格式规范 |

**提取原则**：
- 提取"错误→正确"案例对比
- 标记贡献来源（便于追溯）
- 添加检查清单

## 阶段3：精简压缩

**四删原则**：
1. 删空白"待补充"章节
2. 删重复内容
3. 删冗长描述（改表格）
4. 保留核心案例对比

**目标**：压缩40%-70%，提升信息密度

## 阶段4：逐一验证

**逐篇核对**：
1. 遍历每篇原始文档
2. 列出其关键知识点
3. 核对专题文档中是否覆盖
4. 记录覆盖情况，遗漏则补充

**标准**：每篇文档覆盖率100%

## 阶段5：结构优化

1. **修命名**：统一 `knowledge-YYYYMMDD-type-topic.md`
2. **清归档**：删除旧版/临时文件
3. **同步索引**：主题/标签/时间/文件清单全部更新

## 自检清单

```
□ 双维度分类完成
□ 每个规范有贡献来源
□ 精简后无空白章节
□ 逐一验证覆盖率100%
□ 文件命名统一
□ 4个索引全部同步
```
