# Knowledge Base Manager 使用说明

## 快速开始

### 第一次使用
1. 在Claude中说："整理知识库"
2. 输入你的本地知识库路径（例如：`/Users/yourname/我的知识库/`）
3. 选择整理模式（建议第一次选"全量"）
4. 等待处理完成
5. 下载生成的整理报告和专题文档

### 后续使用
1. 说："整理知识库" 或 "整理最近一周的知识"
2. 确认使用默认路径（或修改）
3. 系统默认增量整理（只处理新文件）
4. 查看生成的报告

---

## 触发方式

### 基础触发
- "整理知识库"
- "管理本地知识"
- "更新知识索引"
- "生成学习报告"

### 带参数触发
- "整理最近一周的知识" → 自动计算日期范围
- "整理学术写作相关的内容" → 只处理学术写作类型
- "全量整理知识库" → 处理所有文件
- "整理1月15日到1月30日的记录" → 指定日期范围

---

## 整理模式

### 增量模式（默认）⭐
- 只处理上次整理后的新文件
- 快速、高效
- 适合定期整理（每周一次）

### 全量模式
- 处理所有文件
- 完整重建索引
- 适合首次使用或大调整后

### 日期范围模式
- 指定起止日期
- 灵活筛选
- 适合专题整理

## 生成的文档

### 必定生成
1. **整理报告-YYYYMMDD.md**
   - 包含摘要和详细分析
   - 位置：`整理报告/`

2. **索引文件**（3个）
   - 主题索引.md
   - 标签索引.md
   - 时间索引.md
   - 位置：`索引/`

### 学术写作专题（如有学术写作内容）
3. **学术写作错误手册.md**
   - 高频错误Top10
   - 改正进展追踪
   - 位置：`专题整理/`

4. **学术写作模式库.md**
   - 可复用写作模板
   - 成功案例
   - 位置：`专题整理/`

### 学术探讨专题（如有学术探讨内容）
5. **思维框架库.md**
   - 可复用思维框架
   - 问题解决模板
   - 位置：`专题整理/`

6. **概念知识网络-[主题].md**
   - 按主题分文件
   - 概念关系图
   - 位置：`专题整理/`

## 常见问题

### Q: 第一次使用应该选什么模式？
A: 选择"全量模式"，建立完整的知识库结构。

### Q: 多久整理一次合适？
A: 建议每周一次，积累5-10个新文档后整理。

### Q: 整理会修改原始文件吗？
A: 不会。只读取原始文件，生成新的整理文档。

### Q: 如果路径输入错误怎么办？
A: 下次整理时可以修改路径，系统会询问是否使用默认路径。

### Q: 专题文档是覆盖还是更新？
A: 采用更新模式，新内容会合并到现有文档中，保留历史版本。

### Q: 可以只整理某个主题吗？
A: 可以，使用类型筛选参数，如"整理学术写作相关的内容"。

## 与conversation-knowledge-extractor的配合

### 完整工作流
```
1. 学习对话
   ↓
2. 触发 conversation-knowledge-extractor
   ↓
3. 生成 knowledge-*.md 文件
   ↓
4. 手动保存到本地知识库
   ↓
5. 积累若干文件（5-10个）
   ↓
6. 触发 knowledge-base-manager
   ↓
7. 生成整理报告和专题文档
   ↓
8. 下载、查阅、复习
   ↓
9. 持续循环
```

### 文件命名一致性
- conversation-knowledge-extractor 生成的文件已符合命名规范
- knowledge-base-manager 可以直接识别和处理
- 保持 `knowledge-YYYYMMDD-topic-XX.md` 格式

## 故障排除

### 文件扫描不到
- 检查路径是否正确
- 检查文件命名是否符合 `knowledge-*.md` 格式
- 检查文件权限

### YAML解析失败
- 检查YAML格式是否正确
- 必须有开头和结尾的 `---`
- 字段名要正确

### 生成文档不完整
- 检查原始文档的元数据是否完整
- 检查是否有足够的标记内容（⭐⚠️💡）
- 增加文档数量（太少可能统计不够）
