## 📝 引言写作模式

### 模式1: 四段式引言 ⭐⭐⭐

**适用场景**: 
- 期刊论文
- 会议长文（6页以上）
- 需要系统阐述背景的论文

**成功率**: 高（应用5次，成功5次）

**结构框架**:
```
第一段: 研究背景（从宽泛到具体）
  - 第1-2句: 领域大背景，建立重要性
  - 第3-4句: 缩小到具体研究方向
  - 第5句: 自然过渡到问题

第二段: 现有研究及局限
  - 第1-2句: 综述相关工作的主流方法
  - 第3-4句: 指出研究空白或局限性
  - 第5句: 强调解决这个问题的必要性

第三段: 本研究动机与目标
  - 第1-2句: 明确提出研究问题
  - 第3句: 说明研究意义
  - 第4句: 阐明研究目标

第四段: 主要贡献与论文结构
  - 第1-3句: 列举核心贡献（通常2-3点）
  - 第4-5句: 简述论文组织结构
```

**成功案例**:
- knowledge-20250115-paper-intro-01.md: 期刊论文引言，审稿人评价"结构清晰"
- knowledge-20250122-paper-intro-02.md: 应用后，从混乱变得有条理

**注意事项**:
⚠️ 第一段不要太长，3-5句为宜，否则读者会失去耐心  
⚠️ 第二段综述要有针对性，不要泛泛而谈  
⚠️ 贡献陈述要具体，避免"we propose a novel method"这类模糊表述  
✅ 可以用数字标记贡献: "First, ... Second, ... Third, ..."

**示例模板**:
```
第一段开头模板:
"In recent years, [broad field] has attracted significant attention 
due to [reason]. Particularly, [specific direction] has emerged as 
a crucial research area..."

第二段过渡模板:
"Despite these advances, existing approaches face several limitations. 
First, ... Second, ... These limitations motivate our research."

贡献陈述模板:
"This paper makes three main contributions. First, we ... 
Second, we ... Third, we ..."
```

## 🧠 概念理解框架

### 框架1: 概念理解三步法 ⭐⭐⭐

**适用场景**: 
理解新的学术概念、理论或方法

**使用频次**: 8次（高频）

**使用步骤**:

**第一步: 是什么（What）**
- 给出精确的定义
- 识别核心特征
- 区分相似概念
- 用一句话总结本质

**第二步: 为什么（Why）**
- 这个概念是如何产生的？（历史背景）
- 为什么需要这个概念？（解决什么问题）
- 它的理论基础是什么？
- 它与其他概念的关系是什么？

**第三步: 怎么用（How）**
- 应用场景是什么？
- 使用步骤/方法是什么？
- 有哪些经典应用案例？
- 使用时需要注意什么？

**示例应用**:

*应用1: 理解强化学习*
- What: 智能体通过与环境交互，最大化累积奖励的学习方法
- Why: 传统监督学习需要标注，而很多场景只有奖励信号（如游戏、机器人）
- How: MDP框架，通过Q-learning等算法实现，应用于游戏AI、机器人控制等

*应用2: 理解注意力机制*
- [详细记录...]

*应用3: 理解Q-Learning*
- [详细记录...]

- 使用后: 平均40分钟，理解深度⭐⭐⭐⭐
- 提升: 效率+200%，深度+100%

**注意事项**:
⚠️ 第一步"是什么"必须确保定义准确，不要一知半解就进入下一步  
⚠️ 第二步"为什么"要真正理解motivation，而不只是记住答案

## 📖 文献解读框架

### 框架1: 论文快速解读五问法 ⭐⭐⭐

**适用场景**: 
快速理解一篇学术论文的核心思想

**使用步骤**（按顺序回答五个问题）:

**问题1: 要解决什么问题？（Motivation）**
- 研究背景是什么？
- 现有方法有什么不足？
- 为什么这个问题重要？

**问题2: 已有哪些尝试？（Related Work）**
- 现有方法A/B/C分别是什么？
- 它们各有什么局限？

**问题3: 本文提出什么方法？（Method）**
- 核心创新点是什么？（用一句话）
- 方法的关键步骤是什么？
- 为什么这个方法可以解决问题？

**问题4: 实验验证了什么？（Experiment）**
- 在什么数据集上测试？
- 主要结论是什么？（定量结果）
- 消融实验说明了什么？

**问题5: 还有什么问题？（Limitation）**
- 方法的局限性是什么？
- 未来可以怎么改进？

**效果**:
- 阅读速度: 从2小时/篇 → 40分钟/篇

## 🔍 问题分析框架

### 框架1: 问题分解树 ⭐⭐⭐

**适用场景**:
面对复杂问题，不知从何入手时

**使用步骤**:
1. 明确总问题
2. 将大问题分解为2-4个子问题
3. 每个子问题继续分解（如果需要）
4. 识别最小可解决单元
5. 逐个击破

**分解原则**:
- MECE原则（相互独立，完全穷尽）
- 每层分解2-4个，不要太多
- 分解到可以直接解决为止

**可视化**:
```
总问题
├─ 子问题1
│  ├─ 子子问题1.1
│  └─ 子子问题1.2
├─ 子问题2
└─ 子问题3
```

**示例应用**:
- 如何提高RL算法的样本效率？
  - 子问题1: 如何减少需要的样本数？
  - 子问题2: 如何更好地利用已有样本？
  - 子问题3: 如何迁移其他任务的知识？
